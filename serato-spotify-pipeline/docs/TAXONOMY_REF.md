## MAPPING STRUCTURE

Each playlist maps to one or more target columns:
- **vibe**: The overall feeling/aesthetic
- **sound**: The sonic character
- **culture**: Cultural origin/style
- **emotional_layers**: Emotional response
- **prominent_instruments**: Key instruments
- **placement**: Role in mix (intro, outro, etc.)

## AGGREGATION BEHAVIOR

When a track appears in multiple playlists:

**Example: Track with ISRC "USUM71234567"**

Input:
- Found in: arabian.csv → vibe: "arabian"
- Found in: set_49_-_bounce.csv → sound: "bouncy"
- Found in: set_55_-_instant_smile.csv → emotion: "euphoric"

Output in enriched file:
```
ISRC: USUM71234567
vibe: "arabian"
sound: "bouncy"
emotional_layers: "euphoric"
```

**Example: Track appearing in 3 vibes**

Input:
- Found in: space.csv → vibe: "space"
- Found in: set_81_-_dreamscape.csv → vibe: "dreamy"
- Found in: set_200_-_epic_rave.csv → vibe: "epic"

Output:
```
ISRC: (same)
vibe: "space|dreamy|epic"
sound: "rave"  (from epic_rave)
```

## USAGE IN SCRIPT

The script reads this mapping from `taxonomy_mapping_final.json` in this format:

```json
{
  "arabian.csv": {
    "vibe": "arabian",
    "culture": "arabian"
  },
  "set_28_-_teardrop.csv": {
    "emotional_layers": "sad"
  },
  "set_84_-_air.csv": {
    "vibe": "air"
  },
  ...
}
```

For each track in each playlist:
1. Check if its ISRC exists in master file
2. If yes, look up playlist name in this mapping
3. Add corresponding column:value pair to enrichment data
4. When all playlists processed, aggregate and combine
